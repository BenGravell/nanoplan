# CommonRoad scenario corpus

The scenario data this repo ships: driving scenarios in the
[CommonRoad](https://commonroad.in.tum.de) 2020a XML format — a lanelet
network, static/dynamic obstacles with full 20 s / 10 Hz state trajectories,
and a planning problem (initial state + goal region) per file, named by
CommonRoad benchmark ID (`ZAM_<Name>-1_<variation>_T-1`; `ZAM` is the
convention for hand-crafted scenarios not tied to a real location).

These files are **original to this repo** (generated by
[`tools/generate_diverse_scenarios.py`](../../tools/generate_diverse_scenarios.py),
validated against the official CommonRoad XSD) and are covered by the
repo's MIT [LICENSE](../../LICENSE) — unlike nuPlan data, they can be
redistributed here and served to the deployed web viewer freely. That's the
point of using the CommonRoad format: an open scenario ecosystem where the
data itself ships with the code.

## How they're used

nanoplan's viewer and batch runner load a simpler JSON format
([`src/scenarios/README.md`](../../src/scenarios/README.md));
[`tools/export_commonroad_scenarios.py`](../../tools/export_commonroad_scenarios.py)
converts any CommonRoad XML into it:

```sh
python3 tools/export_commonroad_scenarios.py scenarios/commonroad scenarios/web
python3 tools/bundle_web_scenarios.py     # -> scenarios/web_bundle.json (web viewer)
cargo run --release --bin batch -- --count 0 --dir scenarios/web
```

Two conversions are checked in at [`scenarios/json/`](../json/) and compiled
into the viewer binary (`ZAM_BrakingLead-1_1_T-1`, `ZAM_CutIn-1_1_T-1`);
the rest ship to the web build via `scenarios/web_bundle.json`.

## Adding real CommonRoad scenarios

The converter handles scenarios from the public CommonRoad database
([commonroad.in.tum.de/scenarios](https://commonroad.in.tum.de/scenarios),
also at [gitlab.lrz.de/tum-cps/commonroad-scenarios](https://gitlab.lrz.de/tum-cps/commonroad-scenarios))
the same way — drop `*.xml` files in here (or anywhere) and convert. Check
the license terms of the specific scenario set you download before
committing it here; sets derived from third-party recordings (e.g. highD)
carry their own terms, while TUM's hand-crafted scenarios are open.

## Regenerating

```sh
python3 tools/generate_diverse_scenarios.py --clean
```

Deterministic per seed — the default settings reproduce this directory
byte-for-byte. `--variations N` generates N randomized variations per
scenario category instead of 1.
